# Contents in this workspace
Inside this workspace is where the project took it's form, this is where multiple prototypes were designed and created and where the final product is located. There are alot of folders and ReadME's in these folders to help the client navigate and understand:
- What the different elements do
- How it works
- The choices that were made
- The research that was done
- The concluding advice

Make sure to **follow all the steps** included and explained in these folders, to run the program correctly and make changes where needed or wanted.

## Main folders
There are two main folders in this workspace:
- models
- prototypes

### Models
In the models folder is where the deffinitve product is located, this folder is set up for easy accessebility and use for the client and future developers. This is also where the code is fully finetuned, connected and functioning as should.

### Prototypes
In the prototypes folder is where the first lines of code were designed, written and tested. These include all the files from the first week of progress to the last. Inside each different folder are **multiple prototypes**, each possible to run from withing the folder itself. That means easy to run scripts with clear steps to follow withing the ReadME's. Besides instructions, **choices and research** were done to come to the final product of the code. 

This is all documented in each folder seperately, where at the bottom an **advice** is also given for the parts. From this advice future developers can continue to write, test and build different improvements neccesary.
