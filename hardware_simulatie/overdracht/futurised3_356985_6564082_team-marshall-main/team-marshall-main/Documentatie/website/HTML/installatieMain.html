Installatie

Installatie- & Configuratiedocumentatie: ROS 2 Jazzy & Gazebo Harmonic

Dit document beschrijft de stappen die nodig zijn om de simulatie-omgeving voor de robot Flip te draaien binnen de Ubuntu 24.04 (Noble) Docker-container.

Snelle start: In de standaard Docker-image zijn alle benodigde pakketten uit Sectie 1 al vooraf geïnstalleerd. Is het gz sim commando niet direct beschikbaar? Sla Sectie 1 over en voer direct de stappen uit Sectie 2 (Omgevingsvariabelen) uit.

Systeempakketten (Alleen bij schone installatie)e

Mocht je vanaf een volledig lege container beginnen of container die niet helemaal voldoet aan de eisen? Check/installeer dan eerst de simulator en de bijbehorende ROS 2- en navigatiepakketten:

apt-get update && apt-get install -y \
    gz-harmonic \
    ros-jazzy-ros-gz \
    ros-jazzy-ros-gz-sim \
    ros-jazzy-slam-toolbox \
    ros-jazzy-rviz2

Essentiële Omgevingsvariabelen & Sourcing

Voordat de commando's en plugins correct werken, moeten de juiste paden actief zijn. Voer de volgende regels uit in je terminal (of voeg ze toe aan je .bashrc / opstartscript):

# 1. Activeer de ROS 2-omgeving en de gekoppelde Gazebo-bridges
source /opt/ros/jazzy/setup.bash

# 2. Geef aan waar Gazebo de robotmodellen van Flip kan vinden
export GZ_SIM_RESOURCE_PATH=/workspace/Futurised/models

Troubleshooting (Veelvoorkomende Problemen)

Als dingen eruit klappen, is het vrijwel altijd een van deze vier problemen:

1. Error: bash: ./run_flip.sh: Permission denied

Linux blokkeert scripts uit veiligheid. Je moet het bestand handmatig uitvoerbaar maken.

Fix: chmod +x run_flip.sh

2. Error: cannot execute: required file not found (bij een .sh script)

Je script heeft onzichtbare Windows-regeleinden (CRLF) in plaats van Linux-regeleinden (LF). Dit gebeurt vaak als je in VS Code op Windows werkt.

Fix: Run dos2unix run_flip.sh om de Windows-tekens te strippen.

3. Error: ImportError: cannot import name 'LaunchDescription' due to circular import

Je hebt je eigen launchbestand letterlijk launch.py genoemd. Python probeert hierdoor de officiële launch-bibliotheek uit jouw eigen bestand te laden.

Fix: Hernoem je bestand naar iets anders, bijvoorbeeld simulation_launch.py.

4. Error: ModuleNotFoundError: No module named 'rclpy'

Je bevindt je in een terminal waarin ROS 2 niet geactiveerd is.

Fix: Run source /opt/ros/jazzy/setup.bash of zorg dat de omgeving geautomatiseerd is.
